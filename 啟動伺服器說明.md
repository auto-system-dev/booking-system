# 如何正確啟動伺服器並查看輸出

## ⚠️ 問題

您嘗試使用 `npm start`，但 PowerShell 無法識別 `npm` 命令（因為 Node.js 沒有在 PATH 中）。

## ✅ 解決方法

### 方法一：使用批次檔（最簡單）⭐ 推薦

**直接雙擊執行：**
- `快速啟動.bat` - 啟動伺服器
- `重新啟動伺服器.bat` - 重新啟動伺服器

這些批次檔已經設定好完整路徑，會自動處理一切！

### 方法二：使用完整路徑

在 PowerShell 中執行：

```powershell
& "C:\Program Files\nodejs\node.exe" server.js
```

### 方法三：暫時加入 PATH

在當前 PowerShell 視窗執行：

```powershell
$env:Path += ";C:\Program Files\nodejs"
npm start
```

（僅在當前視窗有效）

## 📋 啟動後應該看到

伺服器啟動成功後，終端機應該顯示：

```
========================================
🚀 訂房系統伺服器已啟動
📍 網址: http://localhost:3000
📧 Email: cheng701107@gmail.com
========================================

等待請求中...
```

## 🔍 提交表單後應該看到

當您提交表單時，終端機應該顯示：

```
[時間] POST /api/booking
========================================
📥 收到訂房請求
時間: [時間]
請求資料: {...}
========================================

正在發送郵件...
發送給客戶: [您的Email]
使用帳號: cheng701107@gmail.com
✅ 客戶確認郵件已發送
✅ 管理員通知郵件已發送
```

## 🎯 現在請這樣做

1. **使用批次檔啟動**：雙擊 `快速啟動.bat`
2. **或使用完整路徑**：`& "C:\Program Files\nodejs\node.exe" server.js`
3. **確認看到啟動訊息**
4. **提交表單**
5. **查看終端機輸出**，確認郵件發送狀態

## 💡 提示

- 不要使用 `npm start`（因為 npm 不在 PATH 中）
- 使用批次檔或完整路徑
- 保持終端機視窗開啟，這樣才能看到日誌
