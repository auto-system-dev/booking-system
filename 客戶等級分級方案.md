## 🎯 等級評定標準

### 評定方式（三選一或組合）

#### 1. **消費金額優先**（推薦）
- 以累計消費金額為主要評定標準
- 訂房次數為輔助條件（需達到最低次數）
- **優點**：直接反映客戶價值，激勵高消費
- **適用**：中高價位住宿業者

#### 2. **訂房次數優先**
- 以訂房次數為主要評定標準
- 消費金額為輔助條件（需達到最低金額）
- **優點**：鼓勵回購，建立忠誠度
- **適用**：平價住宿、商務旅館

#### 3. **綜合評分制**（進階）
- 消費金額 × 權重 50%
- 訂房次數 × 權重 30%
- 活躍度（最後訂房時間） × 權重 20%
- **優點**：更全面評估客戶價值
- **適用**：大型飯店、連鎖業者

## 📈 等級升級規則

### 自動升級
- ✅ 系統自動計算客戶等級
- ✅ 每次訂房完成後重新評定
- ✅ 達到門檻立即升級（不降級）

### 手動調整
- ✅ 管理員可手動調整等級（特殊情況）
- ✅ 記錄調整原因與時間
- ✅ 支援臨時升級（限時優惠活動）

### 等級保留期
- **建議**：等級保留 12 個月
- 超過 12 個月未訂房，等級降一級
- 超過 24 個月未訂房，降為新會員

## 📊 資料庫設計建議

### 新增欄位（customers 表或 bookings 表擴充）

```sql
-- 建議新增 customers 表（如果尚未建立）
CREATE TABLE customers (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    name VARCHAR(255),
    phone VARCHAR(50),
    member_level INTEGER DEFAULT 1,  -- 1=新會員, 2=銀卡, 3=金卡, 4=鑽石
    total_spent DECIMAL(10,2) DEFAULT 0,
    booking_count INTEGER DEFAULT 0,
    last_booking_date DATE,
    level_updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 或擴充現有 bookings 表，新增會員等級欄位
ALTER TABLE bookings ADD COLUMN member_level INTEGER DEFAULT 1;
```

### 等級計算函數

```javascript
// 計算客戶等級
function calculateMemberLevel(totalSpent, bookingCount) {
    // 方案一：四級制
    if (totalSpent >= 80000 && bookingCount >= 25) {
        return 4; // 鑽石會員
    } else if (totalSpent >= 30000 && bookingCount >= 10) {
        return 3; // 金卡會員
    } else if (totalSpent >= 10000 && bookingCount >= 3) {
        return 2; // 銀卡會員
    } else {
        return 1; // 新會員
    }
}

// 取得等級折扣
function getMemberDiscount(level) {
    const discounts = {
        1: 0,    // 新會員：無折扣
        2: 0.05, // 銀卡：5%
        3: 0.10, // 金卡：10%
        4: 0.15  // 鑽石：15%
    };
    return discounts[level] || 0;
}
```

## 💡 建議採用方案

### 小型民宿/旅館（1-10 間房）
👉 **建議：方案三（簡化三級制）**
- 簡單易管理
- 門檻適中
- 實施成本低

### 中型旅館/飯店（10-50 間房）
👉 **建議：方案一（四級制）**
- 平衡客戶分級
- 激勵效果佳
- 符合專業版功能

### 大型飯店/連鎖業者（50+ 間房）
👉 **建議：方案二（五級制）+ 綜合評分**
- 更細緻的分級
- 可搭配積分系統
- 符合企業版功能

## ⚠️ 注意事項

1. **等級門檻調整**
   - 建議每半年檢視一次
   - 根據實際營運數據調整
   - 避免門檻過高或過低

2. **折扣成本控制**
   - 計算折扣對營收的影響
   - 設定折扣上限（避免虧損）
   - 特殊活動時可臨時調整

3. **等級公平性**
   - 確保升級規則透明
   - 避免手動調整過於頻繁
   - 記錄所有等級變更

4. **客戶溝通**
   - 清楚說明等級規則
   - 定期發送等級狀態通知
   - 升級時發送祝賀訊息
