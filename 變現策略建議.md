# 訂房系統變現策略建議

> 建立日期：2026-02-19

---

## 一、SaaS 訂閱制（最推薦）

將系統包裝成 SaaS 產品，提供給民宿/旅館/飯店業者按月/按年訂閱使用。

| 方案 | 建議月費 | 功能範圍 |
|------|---------|---------|
| **基本方案** | NT$800–1,500/月 | 基本訂房 + 後台管理 + Email 通知 |
| **進階方案** | NT$2,000–3,500/月 | + 綠界金流 + 優惠碼 + LINE Bot |
| **專業方案** | NT$4,000–6,000/月 | + Landing Page + Facebook Pixel + 數據報表 + 優先客服 |

**優勢**：系統已經有多租戶的潛力，只需加入帳號隔離機制即可。

---

## 二、交易抽成模式

對每筆成功的訂房交易收取 **2%–5% 的服務費**。

- 適合不想付月費的小型業者
- 可與訂閱制搭配，例如「基本方案免月費，但抽成 3%」
- 系統已整合綠界金流，技術上很容易實現

---

## 三、客製化建置 + 維護費

以專案制方式幫個別業者建置訂房系統：

- **建置費**：NT$30,000–80,000（依客製程度）
- **年度維護費**：NT$12,000–24,000/年
- 包含：系統部署、品牌客製化、教育訓練

適合預算較高、需要獨立部署的中型旅館。

---

## 四、加值模組銷售

將部分進階功能拆成付費模組單獨販售：

| 模組 | 建議售價 |
|------|---------|
| LINE Bot 整合 | NT$3,000–5,000 一次性 / NT$500/月 |
| 綠界金流串接 | NT$2,000–4,000 一次性 |
| Facebook Pixel + SEO 工具 | NT$1,500–3,000 |
| 自動化 Email 行銷（入住提醒、回饋收集） | NT$1,000–2,000/月 |
| 進階報表與分析 | NT$800–1,500/月 |

---

## 五、白牌授權（White Label）

將系統授權給網頁設計公司或行銷公司轉售：

- 他們以自己的品牌銷售你的系統
- 你收取授權費或每客戶抽成
- 可快速拓展市場，不需自行銷售

---

## 六、開源 + 商業版雙軌模式

- **免費開源**基本版本，吸引開發者社群和流量
- **商業版**提供：進階功能、技術支援、雲端代管
- 類似 WordPress.org（免費）vs WordPress.com（付費）的模式

---

## 七、聯盟行銷 / 整合分潤

- 與旅遊平台（如 Booking.com、Agoda）做 Channel Manager 整合，收取連接費
- 與保險公司合作推「旅遊不便險」，收取推薦佣金
- 與在地體驗業者合作，推薦活動收取分潤

---

## 建議執行優先順序

1. **短期（1-2 個月）**：先以客製建置 + 維護費模式接幾個客戶，驗證市場需求並累積案例
2. **中期（3-6 個月）**：加入多租戶架構，推出 SaaS 訂閱制，搭配交易抽成
3. **長期（6-12 個月）**：發展白牌授權和加值模組生態系，擴大規模

---

## 技術補強項目（走 SaaS 路線需優先開發）

- [ ] 多租戶（Multi-tenant）架構：多業者共用系統但資料隔離
- [ ] 自助註冊 + 引導流程：業者可自行開通帳號
- [ ] 計費系統：自動計算並收取月費/抽成
- [ ] 多物業支持：同一業者可管理多個館別
- [ ] 自訂域名綁定：業者使用自己的網域

## 域名遷移步驟（以 haoding.tw 為例）

購買域名後，依以下順序執行：

### 第一階段：DNS 設定
1. 將 `haoding.tw` 加入 Cloudflare，Nameserver 指向 Cloudflare
2. 設定 DNS：`haoding.tw` 和 `www` CNAME 指向 Railway 網址

### 第二階段：Railway 綁定域名
3. Railway Settings → Custom Domain → 新增 `haoding.tw` 和 `www.haoding.tw`

### 第三階段：R2 Custom Domain
4. R2 Bucket Settings → Custom Domains → 新增 `images.haoding.tw`
5. 更新 Railway 環境變數 `R2_PUBLIC_URL=https://images.haoding.tw`

### 第四階段：更新程式碼中的網址引用
6. server.js 中所有 Railway 網址替換為 `haoding.tw`
7. 綠界後台更新付款回傳 URL
8. LINE Bot Webhook URL 更新
9. 資料庫中舊 R2 URL 批次更新

### 第五階段：SSL 與驗證
10. Cloudflare SSL 設為 Full 或 Full (Strict)
11. 全面測試（前台、後台、訂房流程、付款、Email、LINE Bot）

預估工作時間：約 1.5 小時（不含 DNS 生效等待時間）

## 多租戶域名方案

### 方式一：子域名（優先實作）

每個業者分配子域名，例如：
- `sunshine.haoding.tw`（陽光民宿）
- `shanhai.haoding.tw`（山海旅館）

技術實作簡單，用 Cloudflare 萬用憑證（Wildcard SSL）一張搞定所有業者。

### 方式二：業者自訂域名（進階方案加值功能）

業者用自己的域名指向系統，例如：
- `booking.sunshine-inn.com.tw`

業者需在自己的 DNS 新增 CNAME 紀錄指向系統。
SSL 憑證可用 Cloudflare for SaaS（免費支援 100 個自訂域名）或 Let's Encrypt。

### 建議實作順序

| 優先順序 | 項目 | 時機 |
|---------|------|------|
| 1 | 子域名 `業者.haoding.tw` | 多租戶上線時就做 |
| 2 | 業者自訂域名 | 有業者提出需求時再做 |
